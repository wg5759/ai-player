# AgentPlay 后续工作清单

更新时间：2026-07-19。本文只记录尚未达到“稳定交付”标准的工作；代码入口或按钮存在不等于完成。

## 当前阶段：AI 文档工作台

最终交互目标（用户已确认）：一个“打开”入口能打开所有文件，一个 AI 对话窗处理所有问题；功能能藏在技术后面就绝不暴露成面板和按钮，首页保持播放器的简洁。现有分离面板（文档工作台、拉片工作室、模型中心）逐步收编为同一对话流的能力，不再增加新独立入口。实现手段坚持“超级智能化”：可选组件（本地模型、OCR、格式转换引擎、更大模型等）一律由应用按任务自动下载、校验、安装，沿用组件包清单+SHA-256+断点续传模式，不让用户自己找；模型接入做到粘贴 Key 即自动识别、测速、列模型。桌面安装程序之外，调研 Web/PWA 版承接 AI 对话与文档处理（浏览器内完成，免安装、免代码签名），播放、本地AI推理和系统集成为桌面专属能力。

首个可用闭环已经进入源码：

- 文字输入和语音输入共用同一套任务流程；
- 受控选择文件，原文件默认不覆盖；
- 文本/DOCX生成与基础转换；
- XLSX/CSV清理、去重、明确公式写入，以及自然语言公式规划；
- PPTX新建；
- PDF合并、按页拆分、文字层提取（无文字层的扫描件会明确提示待 OCR）；
- 云模型发送文件正文前逐次授权；
- Windows资源管理器独立“用 AgentPlay 智能处理”右键动词，与“播放”动词分开，文档不再误入播放器；
- 结果文件、历史记录、取消入口和自动化回归测试。

该阶段仍需继续深化：

1. DOCX已有复杂版式的无损修改：确定性查找替换（跨run文字）、修订模式（w:ins/w:del）、锚点/序号段落插入、批注、文末追加均已本地无损落地（样式/表格/图片/页眉页脚不动）；行内样式全保真与任意位置段落级样式继承仍需继续。
2. 已有PPTX的页面级编辑：整稿/单页文字替换、删除指定页（关系与备注页全套清理）、新增页（沿用现有母版版式、标题+要点、指定位置）已本地无损落地（母版/版式/主题/备注/媒体逐字不动）；图表与动画参数化编辑仍需继续。
3. 扫描PDF OCR：无文字层 PDF 已自动回退到 Windows 系统 OCR（应用内 Chromium 按页栅格化+WinRT 中文识别，零下载零配置），提取结果标注扫描件与误差提示；更高精度 OCR 引擎（RapidOCR 组件包）、表格结构恢复、双语重排和可靠脱敏仍需继续。
4. Excel公式：本地安全子集引擎（白名单函数、零eval）已落地——写入前拦截循环引用（含区域覆盖目标列），写入后按真实数据重算抽验并报告样本值与错误行（如#DIV/0!），不支持的函数（如VLOOKUP）写入但明确标注未抽验；图表、透视表和跨工作簿引用仍需继续。
5. DOCX/PPTX/XLSX与PDF之间的高保真转换：→PDF 方向已落地——指令含"高保真/原样/保真"即调用本机 Office 引擎（Word/Excel/PowerPoint COM，宏强制禁用、超时只清理新进程），未检测到引擎时明确故障关闭并提示普通转换；PDF→DOCX 反向高保真与 LibreOffice/WPS 适配仍需继续。
6. 多文件联合任务：已落地——指令提及两种以上格式且有"和/+/一套"等成套意图时，一次模型调用产出成套 JSON（Word报告/Excel分析/PPT汇报/PDF交付/MD/TXT 任意组合），各格式确定性渲染并统一命名"源名-AgentPlay处理版-报告/分析/汇报/交付"，历史记录全套；跨文件数据一致性校验仍需继续。
7. Windows正式安装包中的真实界面、文件重开和大文件压力验收。
8. 本地AI组件改为应用内下载：只保留标准版一个安装包，模型接入中心提供带进度、断点续传和SHA-256校验的组件下载（代码与自动化测试已完成）。组件包已在 release/local-ai-pack/ 生成，待上传到 GitHub Release 的 local-ai-pack-v1 标签后做真实下载验收；验收通过前保留 build:electron:local-ai 作为回退。

## AI高阶能力

1. 多模态拉片和深度解剖需要用实际已配置的视觉模型、字幕和人工镜头证据做端到端质量验收。
2. AI新镜头、旁白、配音、字幕包装、音乐混剪已具备工作流和渲染入口，仍需按不同模型与失败场景做真实任务验收。
3. 外语无字幕视频实时识别、翻译和双语字幕；有字幕时把译文稳定排在原字幕下方。
4. 网页、新闻和社交网站的全局翻译需要独立的浏览器扩展/受控浏览器方案，不能用桌面播放器按钮冒充全局能力。
5. 文档工作台和视频工作台打通：视频/网页 → 字幕/正文 → 拉片表 → 报告 → PPT/PDF → 新成片。
